# **General Patterns from All Trajectories**

### **1. Useless Actions When Target Is Far**
- In ALL trajectories, the agent repeatedly takes actions on "wall" cells, usually (0,0), doing nothing while the target is not adjacent or close.
    - **Properties:**
      - Target is far away (large manhattan distance to agent)
      - Action cell contains "wall"
    - **Hint:** *When the target is not nearby, the agent waits or does nothing (e.g., clicking on a wall cell)*.

### **2. Transition to Purposeful Movement as Target Gets Near**
- Once the target is **near enough** (in the same row, column, or reachable), the agent begins moving towards it.
    - The agent starts taking purposeful actions (using directional arrows, stepping onto empty cells, etc.).
    - **Properties:**
      - Distance to target has decreased sufficiently.
      - Pathways to target are available (no wall directly between agent and target).
    - **Hint:** *Start moving towards the target only when it's within a reachable range (i.e., the agent stops waiting when the target gets close).*

### **3. Use of Directional Arrow Tiles for Movement**
- The agent makes use of special tiles: **right_arrow, up_arrow, left_arrow, down_arrow**.
    - The agent clicks directional arrows repeatedly to move in the desired direction.
    - When arrows are used, they steer the agent towards the target, often over multiple steps.
    - **Properties:**
      - Directional arrows are available at consistent locations.
      - The desired movement direction matches the available arrow (e.g., to move up, click up-arrow).
      - The nearest arrow in the *correct direction relative to agent* is chosen.
    - **Hint:** *To move in a direction, use the directional arrow tile for that direction, even if it's far away from the agent.* 

### **4. Drawing a Trail When Clicking on Cells In Target's Path**
- In some cases, the agent clicks an empty cell well away from itself, and a "Drawn" tile appears.
    - **Properties:**
      - Action cell is empty and is on the potential path the target might take, or is in the direction of the target.
      - These drawn cells seem to play a role in steering, blocking, or "marking" for the target.
    - **Hint:** *Click empty grid cells near the target or along the path as the target approaches these locations.*

### **5. Adjacent-to-Target Behavior**
- When the agent is immediately adjacent to the target, the agent moves into position to capture or stay close.
    - **Properties:**
      - Agent is one move away (manhattan distance = 1) from the target.
      - Use directional arrow or move onto adjacent empty cell to reach/meet the target.
    - **Hint:** *When adjacent to the target, take final move to reach/capture, often via up_arrow or another directional tile.*

### 1. START Production (STRICT)
- START is a normal nonterminal following uniqueness rules.
- Only primitives with `"pcfg_insertion.nonterminal": "START"` may appear in START.
- START must have **≥2 alternatives**.
- Each START alternative must:
  - Call a START-owned primitive
  - Contain at least one argument that is CONDITION or LOCAL_PROGRAM
  - End with `a, s`
- No lambdas in START.
- Base-case primitives may not be START-owned.

START alternatives must be constructed mechanically:
1. Take a primitive whose owning nonterminal is `"START"`.
2. Read its args list in order.
3. Replace each arg with the corresponding NT name.
4. Emit: `"PRIM(NT1, NT2, ..., a, s)"`.

---

### 2. LOCAL_PROGRAM Production (STRICT)
- Only primitives with `"pcfg_insertion.nonterminal": "LOCAL_PROGRAM"` may appear in LOCAL_PROGRAM.
- LOCAL_PROGRAM must contain:
  1. The literal alternative: `"CONDITION"`
  2. ≥1 recursive lambda alternative using LOCAL_PROGRAM arguments.
- Format for lambda alternatives:
  `"lambda cell, obs: PRIM(ARG1, ARG2, ..., cell, obs)"`.
- LOCAL_PROGRAM may not include base-case primitives.
