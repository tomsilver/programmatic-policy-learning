You are writing Python code that generates boolean feature templates for Logical Programmatic Policies (LPP) in Generalization Grid Games (GGG).

Goal:
Write a single self-contained Python script that defines a feature-template generator for discrete grid games. The generator should return a compact library of interpretable boolean predicate templates f(s, a) that are useful for distinguishing expert clicks from plausible non-expert clicks.

Environment assumptions:
- Observation s is a 2D grid/list of token values.
- Action a is a clicked cell coordinate: a = (row, col).
- Action coordinates may be ordinary Python ints or numpy integer scalars.
- Next state s' is shown in the demonstrations only for understanding transitions.
- Features must depend only on s and a.
- Features are intended for downstream logical policy learning, so they must be interpretable boolean predicates.
- The same prompt should work across grid games such as TwoPileNim, Chase, StopTheFall, ReachForTheStar, and CheckmateTactic.

Do not keep templates solely because they are easy to define from the raw grid.
