Use BOTH the demonstrations and the structural hints.

First, verify that the structural hints are consistent with the demonstrations by contrasting chosen actions with plausible alternative clicks in the same state.

If a hint appears unsupported by the demonstrations, ignore it.
If the demonstrations reveal a consistent structural distinction not captured by the hints, include templates to capture it.

Then create templates that express the necessary distinctions at multiple spatial scales (cell-level, line-level, region-level, and whole-grid summaries). Include some state-only templates that ignore `a`.

Features must:

## FEATURE REQUIREMENTS

1) Signature:
   def fK(s, a):

2) Safety:
   Must validate action using try/except and bounds check exactly as specified.

3) No imports.
   No external dependencies.
   Use only Python built-ins.

4) Structure:
   You may define small inner helper logic inside each feature.
   Keep functions readable and short.

5) Generality:
   - No hard-coded coordinates.
   - No raw token comparisons.
   - Only use ${TOKEN}, ${DRs}, ${K} placeholders.

6) Diversity:
   - Avoid near-duplicate templates.
   - Cover:
     - action-cell properties
     - vertical and horizontal structure
     - directional scans
     - nearest-hit logic
     - row/column aggregation

   Do NOT include both a predicate and its obvious direct complement as separate features 
   (e.g., is_X vs is_not_X, etc).
   It is OK to use `not` inside a feature, but do not create a separate feature whose only 
   purpose is the negation of another feature.

8) Naming:
   id and name must be "f1", "f2", ..., "f100".
   source must be a valid Python function string.
   Newlines must be escaped as \\n inside JSON.
